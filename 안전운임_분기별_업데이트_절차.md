# 안전운임 데이터 분기별 업데이트 절차

## 전체 흐름

```
국토부 원본 Excel → 변환 스크립트 실행 → 편집본 생성 → Python 파싱 → SQL 분할 파일 생성 → D1 DB 삽입
```

---

## 사전 준비 (최초 1회만)

- Node.js, Python 설치 확인
- `pip install pandas openpyxl` 설치
- Wrangler 로그인: `npx wrangler login`

---

## STEP 1. 국토부 원본 파일 편집

1. 국토부 고시 원본 Excel 파일 다운로드
2. 변환 스크립트 실행하여 편집본 생성
3. 편집본 확인 사항:
   - 사용할 시트만 남기고 나머지 삭제 (사용 항구: 부산신항, 부산북항, 인천신항, 인천항, 의왕ICD)
   - 시트명: 한글 항구명 그대로 유지
   - 컬럼명 (1행): `sido`, `sigungu`, `eupmyeondong`, `distance`, `40ft_round`, `20ft_round`
4. 파일명: `SafeFreightRate_YYYYMMDD.xlsx` (고시 날짜 반영)
5. 파일을 `freight-rate-app` 폴더에 복사 (기존 파일 교체)

---

## STEP 2. import_data.py 파일명 수정

`import_data.py` 상단의 파일명을 새 파일명으로 수정:

```python
EXCEL_FILE = "SafeFreightRate_YYYYMMDD.xlsx"  # ← 날짜 변경
```

---

## STEP 3. Python 파싱 및 SQL 파일 생성

`freight-rate-app` 폴더에서 실행:

```bash
python import_data.py
```

실행 결과:
- `data_init.sql` (기존 데이터 초기화 파일)
- `data_001.sql` ~ `data_NNN.sql` (500건씩 분할된 데이터 파일)
- 터미널에 실행할 명령어 목록 출력됨 → 마지막 파일 번호(NNN) 확인

---

## STEP 4. 로컬 DB 업데이트

### 4-1. 스키마 적용 (DB 구조 변경이 있을 경우만)
```bash
npx wrangler d1 execute freight-rate-db --local --file=./schema.sql
```

### 4-2. 기존 데이터 초기화
```bash
npx wrangler d1 execute freight-rate-db --local --file=./data_init.sql
```

### 4-3. 데이터 일괄 삽입 (PowerShell)
```powershell
for ($i=1; $i -le NNN; $i++) { $f = "data_{0:D3}.sql" -f $i; Write-Host "실행: $f"; npx wrangler d1 execute freight-rate-db --local --file="./$f" }
```

> ⚠️ `NNN` 부분은 STEP 3에서 확인한 실제 생성 파일 수로 변경

### 4-4. 데이터 확인
```bash
npx wrangler d1 execute freight-rate-db --local --command="SELECT port, COUNT(*) as 건수 FROM freight_rates GROUP BY port"
```

## 파일 구조 참고

```
freight-rate-app/
  src/
    App.tsx          ← 프론트엔드 (React)
    worker.ts        ← API 서버 (Hono)
    App.css          ← 스타일
    index.css        ← 전역 스타일
  import_data.py     ← 데이터 파싱 스크립트 (분기별 실행)
  schema.sql         ← DB 스키마 (구조 변경 시만 수정)
  wrangler.toml      ← Cloudflare 설정
  SafeFreightRate_YYYYMMDD.xlsx  ← 편집본 Excel (분기별 교체)
```

## 할증 요율 변경 시

`src/App.tsx`의 `calcRate()` 함수에서 수정:

| 항목 | 현재 요율 | 코드 |
|------|----------|------|
| 편도 운임 | -50% | `rate *= 0.5` |
| 위험물 할증 | +30% | `rate *= 1.3` |
| 중량초과 | +10%/톤 | `rate *= 1 + overweight * 0.1` |

> ⚠️ 요율 변경 후 반드시 로컬 테스트 → Cloudflare 재배포 필요
