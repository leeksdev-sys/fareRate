# 안전운임 데이터 분기별 업데이트 절차

## 전체 흐름 (현재 표준)

```
국토부 원본 Excel → convert_freight_data.py(CSV 생성) → import_data.py(SQL 분할 생성)
→ update.ps1(local/remote) 실행 → D1 반영 + (remote 시) 빌드/배포
```

---

## 사전 준비 (최초 1회)

- Node.js, Python 설치
- Python 패키지 설치: `pip install pandas openpyxl`
- Wrangler 로그인: `npx wrangler login`
- 운영 시크릿 확인:
  - `npx wrangler secret list`
  - `ADMIN_PASSWORD`, `ADMIN_TOKEN_SECRET` 존재해야 함

없으면 등록:

```bash
npx wrangler secret put ADMIN_PASSWORD
npx wrangler secret put ADMIN_TOKEN_SECRET
```

---

## STEP 1. 원본 데이터 준비

1. 국토부 고시 원본 Excel 파일 확보
2. `convert_freight_data.py`가 있는 데이터 폴더에 원본 파일 배치
3. 파일/시트 구조는 기존 변환 스크립트 입력 형식 유지

> `update.ps1`가 상위 폴더에서 `convert_freight_data.py`를 자동 탐색하므로, `import_data.py` 내부 파일명을 수동 수정할 필요가 없습니다.

---

## STEP 1.5 사전점검 (원클릭)

분기 업데이트 시작 전에 아래 1회 실행:

```powershell
.\precheck.ps1
```

점검 항목:
- 필수 파일 존재 여부 (`update.ps1`, `deploy.ps1`, `schema.sql`, `data_init.sql`, `wrangler.toml` 등)
- `data_init.sql` 시그니처 확인
- `db_sync` 최신 업체 백업 SQL 존재 여부
- 잔여 분할 SQL 파일 여부
- Python/Node/NPX 실행 가능 여부
- `wrangler` 시크릿(`ADMIN_PASSWORD`, `ADMIN_TOKEN_SECRET`) 존재 여부

`Errors: 0`일 때만 다음 단계 진행 권장.

---

## STEP 2. 로컬 검증 업데이트 (권장 선행)

`freight-rate-app` 폴더에서 실행:

```powershell
.\update.ps1 local
```

실행 시 자동 처리:
- Excel → CSV 변환
- CSV → `data_001.sql` ~ `data_NNN.sql` 생성
- `data_init.sql` 실행(테이블 드롭)
- `schema.sql` 재생성
- 분할 SQL 로드
- `db_sync/companies_local_data_*.sql` 최신 백업 복원
- 건수 검증
- 분할 SQL 파일(`data_###.sql`) 자동 삭제

---

## STEP 3. 로컬 동작 확인

```bash
npm run build
```

필요 시 실행 확인:

```bash
npx wrangler dev
npm run dev
```

---

## STEP 4. 운영 반영 (원격 DB + 배포)

```powershell
.\update.ps1 remote
```

원격 모드에서 자동 처리:
- 원격 D1 드롭/스키마 재생성/데이터 로드
- 업체 백업 복원
- 건수 검증
- 프론트 빌드
- `wrangler deploy`
- 분할 SQL 파일 자동 삭제

---

## 대체 배포 스크립트 (선택)

데이터 생성은 이미 끝난 상태에서 배포만 할 때:

```powershell
.\deploy.ps1 local
.\deploy.ps1 remote
.\deploy.ps1 all
```

또는 Bash 환경:

```bash
bash deploy.sh local
bash deploy.sh remote
bash deploy.sh all
```

## 주의사항

- `data_init.sql`은 `freight_rates`, `companies`를 DROP 합니다. 단독 실행 금지.
- 반드시 `schema.sql` 및 데이터 로드가 이어지는 스크립트(`update.ps1`, `deploy.ps1`, `deploy.sh`)로만 실행.
- 수동 정리 시 `data_init.sql`은 삭제하지 말고 유지(스크립트에서 사용).
- 배포/업데이트 전에는 가능하면 `precheck.ps1`을 먼저 실행.

## 할증 요율 변경 시

`src/App.tsx`의 `calcRate()`에서 조정:

| 항목 | 현재 요율 | 코드 |
|------|----------|------|
| 편도 운임 | -50% (왕복 노선에만 적용) | `if (oneWay && !isOneWayRoute) rate *= 0.5` |
| 위험물 할증 | +30% | `rate *= 1.3` |
| 중량초과 | +10%/톤 | `rate *= 1 + overweight * 0.1` |

요율 변경 후 `npm run build` 확인 및 재배포 필수.
